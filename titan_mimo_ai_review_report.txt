1. ARCHITECTURE OVERVIEW
--------------------------------------------------------------------------------
Model Name: Titan MIMO 650M
Base Architecture: Mamba S6 (Selective State Space Model)
Parameters: ~650 Million
Hidden Size (d_model): 1024 (Assumed scaling based on 650M class)
Sequence Length: 1024

Key Architectural Features:
* Continuous/Discrete Hybrid Backbone: Token embeddings, LayerNorms, and LM head remain continuous (FP32) to allow smooth gradient flow for representation mapping. The core Mamba SSM projections (in_proj, out_proj) for all layers are wrapped in the `PrimeDiscrete` optimizer.
* MIMO (Multi-Input Multi-Output) Arms: The standard Mamba backbone is bisected. After the mid-point, a DYNAMIC DOMAIN ROUTER explicitly routes the forward pass through one of four expert Mamba blocks (Chat, Math, Code, Tool) based on the latent context of the prompt before rejoining the final backbone layers. 
* Prime Harmonic LUT: All discrete weights are stored as `uint16_t` indices pointing to a shared 65,536-element float32 Look-Up Table (LUT) populated with prime harmonic fractions. 

2. TRAINING METHODOLOGY (PRIME OPTIMIZER)
--------------------------------------------------------------------------------
Instead of AdamW/SGD updating FP32 parameters, this model uses the `ContinuousVoteOptimizer` to execute training via integer voting.
* Gradient Sign Accumulation: Gradients for the discrete `PrimeLinear` weights are converted to their sign (+1 or -1).
* Pressure Buffer: These signs accumulate in a `int16` vote buffer across micro-batches.
* Supermajority Threshold: A parameter bit is only allowed to "flip" (shift its LUT index up or down by 1) if the absolute accumulated vote pressure exceeds a dynamic threshold.

3. HOMEOSTATIC PI CONTROLLER (V2)
--------------------------------------------------------------------------------
Every single `PrimeLinear` layer (including the new domain router) is equipped with an autonomous Proportional-Integral (PI) Controller operating in log-space.
* Mechanism: The user configures a `target_flip_rate` per layer (e.g., 10% for backbone, 13% for MIMO arms, 16% for the router). Every 50 steps, the PI controller compares the layer's Exponential Moving Average (EMA) flip rate against the target. It adjusts the log-threshold of the supermajority gate (Kp=0.5, Ki=0.01) to dynamically increase or decrease resistance. 
* Result: The flip rate across all 650M parameters is strictly maintained, completely absorbing massive gradient shocks (like MoE routing anomalies) without causing catastrophic forgetting.

4. CRITICAL PROBLEMS & FIXES DETAILED
--------------------------------------------------------------------------------
### A. The VRAM OOM on Inference
* Fix: Decoupled inference. The training loop must be explicitly paused (`kill <PID>`) to clear VRAM before evaluating the model natively on the GPU.

### B. The Artificial Padding Loss Deflation Bug
* Problem: At Step 217,700, the model aggressively optimized its parameter space to predict padding tokens (`<|endoftext|>`), destroying its ability to generate English.
* Fix: Updated the forward pass to use `F.cross_entropy(logits, labels, ignore_index=0)`.

### C. MoE OOD Blending Explosion (Phase 4)
* Problem: Upon unfreezing the dynamic router, the `F.softmax` output fractional routes (e.g., 0.3 Chat, 0.5 Math). Blending the output of the experts created an Out-Of-Distribution (OOD) representation that exponentially exploded the downstream Mamba selective-scan algorithms, pushing the loss into the quadrillions.
* Fix: Replaced Softmax with `F.gumbel_softmax(hard=True)` (Straight-Through Estimator). This forces the forward pass into a strict 1-hot route (protecting downstream layers), while preserving soft routing loss gradients for backpropagation.

### D. Unnormalized Residual Variance (Phase 4)
* Problem: The routing loss itself evaluated to over 2.6 million because the router was reading the raw, unnormalized residual stream `x` from the middle of the network, which possessed massive variance.
* Fix: Applied `F.layer_norm` to the detached residual state `x` prior to classification, strictly bounding the logits and instantly stabilizing the routing loss to ~1.38.

### E. Gumbel Hallucination & Padding Masking
* Problem: Gumbel noise during inference mode caused erratic routing. Furthermore, the router was wasting gradients trying to classify the domain of padding tokens.
* Fix: Implemented a greedy `torch.argmax` bypass for `model.eval()`. Masked out padding tokens in the routing loss using `ignore_index=-100`.
